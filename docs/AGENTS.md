# Docs Agent Guide

## Scope

`docs` contains architecture, operations, OpenAPI, LSP, hooks, and product/integration documentation.

## Rules

- Update docs when API contracts, startup/deployment flows, architecture boundaries, or compliance wording changes.
- Prefer concise, durable documents over progress logs.
- Keep docs aligned with root `README.md`, `AGENTS.md`, `CODEMAP.md`, and `docs/ARCHITECTURE.md`.
- Do not document secrets or production credentials.
