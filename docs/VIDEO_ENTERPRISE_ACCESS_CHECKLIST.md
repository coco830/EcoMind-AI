# 企业侧视频接入清单

## 1. 使用目标

本清单用于企业视频接入前的资料收集、接口确认、网络协调和验收准备。

适用当前阶段：

- EcoMind-AI 已具备视频台账、安装验收和告警联动能力
- 企业侧暂时还没有正式提供 `VMS / GB28181 / RTSP / 回调接口`
- 平台先收集资料、建设台账，后续再平滑切换到真实视频联调

---

## 2. 必收字段

### 2.1 协议类型

至少确认以下其一：

- `GB28181`
- `RTSP`
- `ONVIF`
- `HTTP/HTTPS 预览链接`
- `企业 VMS 平台跳转`

需要企业明确：

- 实际采用哪种协议
- 是否支持多协议并存
- 是否由运营商平台/州市平台/企业自建平台统一转发

### 2.2 通道编码

每个视频点位至少提供：

- 设备编码
- 通道编码
- 通道名称
- 对应排口/站房/采样点名称
- 是否主视角/辅助视角

建议补充：

- 摄像头品牌型号
- 安装位置描述
- 机位朝向

### 2.3 预览地址

若支持直接预览，需提供：

- 预览 URL
- 鉴权方式
- 是否有有效期
- 是否需携带 token / appKey / sign
- PC 端是否可直接打开

### 2.4 回放地址

若支持录像回放，需提供：

- 回放 URL
- 时间参数格式
- 支持的时间跨度
- 是否支持按通道回放
- 是否支持片段截取

### 2.5 回调方式

若企业平台可主动推送事件，需确认：

- 回调协议：`HTTP / HTTPS / MQ / Webhook`
- 回调地址白名单要求
- 回调鉴权方式
- 回调重试机制
- 回调事件类型

建议优先支持的事件：

- 视频离线
- 通道上线
- 抓拍事件
- AI 识别事件
- 录像片段生成完成

### 2.6 网络要求

至少确认：

- 网络承载方式：`公网 / 专线 / VPN / 政务外网 / 企业内网`
- 是否提供固定 IP
- 是否需要我方固定出口 IP 加白
- 需要开放的端口
- 上下行带宽
- 是否允许跨网访问
- 是否需要 HTTPS 证书

## 3. 建议收集表

| 分类 | 字段 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 企业信息 | 企业名称 | 是 | 与平台组织一致 |
| 企业信息 | 联系人/电话 | 是 | 视频平台协调人 |
| 点位信息 | 监测设备名称 | 是 | 与数采仪设备对应 |
| 点位信息 | 点位类型 | 是 | 站房 / 排口 / 手工采样点 |
| 点位信息 | 摄像头名称 | 是 | 平台展示名称 |
| 点位信息 | 通道编码 | 是 | 平台对接主键之一 |
| 接入方式 | 协议类型 | 是 | GB28181 / RTSP / ONVIF / 链接 |
| 接入方式 | 接入来源 | 是 | 企业平台 / 运营商平台 / 州市平台 |
| 链接信息 | 预览地址 | 否 | 无真实流时可先留空 |
| 链接信息 | 回放地址 | 否 | 无回放能力时留空 |
| 鉴权信息 | token / appKey / sign 规则 | 否 | 由企业提供 |
| 回调能力 | 回调地址 | 否 | 企业支持主动推送时填写 |

| 网络信息 | 端口要求 | 否 | 如 SIP / RTP / RTSP / HTTPS |
| 交付信息 | 勘点负责人 | 否 | 可先由环保管家维护 |

| 交付信息 | 验收负责人 | 否 | 业主或项目负责人 |

## 4. 不同协议的重点确认项

### 4.1 GB28181

重点确认：

- 国标版本
- SIP 服务器地址
- SIP 端口
- 设备编码 / 通道编码
- 注册方式
- 心跳周期
- 是否需要级联
- 实际媒体流端口范围

### 4.2 RTSP

重点确认：

- RTSP 地址
- 用户名密码
- H.264 / H.265 编码方式
- 主码流 / 子码流
- 分辨率与帧率
- 公网或专网可达性

### 4.3 企业 VMS / 平台跳转

重点确认：

- 是否提供统一预览 URL
- 是否支持免登或单点登录
- token 有效期
- 是否支持按通道打开
- 是否支持回放深链
- 是否支持抓拍或录像片段接口

---

## 5. 验收前最少联调项

在正式接入前，建议至少完成以下 8 项确认：

1. 平台中已建立视频通道台账，且与设备绑定正确
2. 每个点位已明确建设阶段和安装位置
3. 企业已提供协议类型和通道编码
4. 企业已明确预览/回放是否可用
5. 网络承载方式和加白要求已确认
6. 回调方式已明确，至少知道“有/无”
7. 至少 1 个点位完成预览联调
8. 至少 1 条告警链路可人工验证“告警 -> 视频联动事件”

---

## 6. 当前阶段的执行建议

在没有真实 VMS 的情况下，建议按以下顺序推进：

1. 先在 EcoMind-AI 中维护视频接入台账和安装验收台账
2. 使用平台内置 demo 数据完成页面、权限、流程和联动测试
3. 与企业沟通本清单中的协议、通道、链接、回调和网络资料
4. 选 1 家企业、1 个排口做首个真实联调试点
5. 再逐步扩展到多企业、多点位
