# HJ212 协议技术规范摘要 (兼容 2017/2025 版)
> 本文档综合了 HJ 212-2017 和 HJ 212-2025 标准，旨在开发兼容双版本的解析器。
## 1. 通信包结构 (通用)
格式：`##<DataLen><DataSegment><CRC><\r\n>`
- **包头**：`##` (2字节)
- **DataLen**：4字节十进制字符串，表示数据段长度（例如 "0120"）。
- **DataSegment**：变长字符串，包含具体指令和数据。
- **CRC**：4字节十六进制字符串，是对 DataSegment 进行 CRC16 校验的结果。
- **包尾**：`\r\n` (ASCII 13, 10)

## 2. 数据段 (DataSegment) 结构
格式：`QN=...;ST=...;CN=...;PW=...;MN=...;Flag=...;CP=&&...&&`

### 关键字段定义
- **QN**: 请求编码，精确到毫秒的时间戳 (YYYYMMDDHHMMSSmmm)。
- **ST**: 系统编码。
  - `31`: 环境污染源 (2017/2025通用)
  - `32`: 地表水 (2017/2025通用)
  - `22`: 空气质量 (2017/2025通用)
  - `42`: 废气工况监控 (2025新增)
  - `44`: 设施用电监控 (2025新增)
- **CN**: 命令编码。
  - `2011`: 实时数据上报
  - `2051`: 分钟数据上报
  - `2061`: 小时数据上报
  - `9013`: 心跳包/通知应答
  - `9014`: 数据应答
- **MN**: 设备唯一标识 (24位字符)。
- **Flag**: 标志位 (8位整数，0-255)。**解析器需进行位运算判断版本：**
  - **Bit 0 (LSB)**: `0`=不应答, `1`=应答 (Ack)
  - **Bit 1**: `0`=无包序号, `1`=有包序号 (Split)
  - **Bit 2-7 (Version)**: 协议版本号
    - 若 `(Flag >> 2) == 1` : **HJ 212-2017** (处理逻辑：纯明文解析)
    - 若 `(Flag >> 2) > 1` (通常为2或更高): **HJ 212-2025** (处理逻辑：检查加密位，可能需 SM4 解密)
    - *注：2025版可能复用高位或增加扩展字段指示加密，具体依据 2025 附录 A 判断。*

## 3. CP 数据区 (Content Packet) & 加密逻辑
格式：`&&数据区&&`

### 处理流程
1.  **提取 CP 内容**：获取两个 `&&` 之间的字符串。
2.  **判断加密**：
    - **如果是 2017 版**：**永远不加密**，直接进入步骤 3。
    - **如果是 2025 版**：检查 Flag 或配置。如果指示加密，内容为 Hex 字符串。需使用 **SM4-ECB** 算法 + 预置密钥进行解密，解密后得到明文，再进入步骤 3。
3.  **解析参数**：
    - 键值对结构：`Key=Value`
    - 多个参数分隔：`;`
    - 同参数属性分隔：`,`
    - 示例：`w01001-Rtd=7.5,w01001-Flag=N;w01002-Rtd=23.5`

## 4. 常用参数编码 (映射表)
解析器应建立字典映射，不要硬编码。

### 污染物 (w/a 系列) - 2017/2025 通用
- **w01001**: pH值
- **w01018**: 化学需氧量 (COD)
- **w21003**: 氨氮
- **w21011**: 总磷
- **w21001**: 总氮
- **a01011**: 烟气流速
- **a01012**: 烟气温度
- **a34013**: 烟尘(颗粒物)
- **a21026**: 二氧化硫 (SO2)
- **a21002**: 氮氧化物 (NOx)

### 新增参数 - 2025 专用
- **dxxxx**: 用电参数 (如 `d10001` 总有功功率)
- **pxxxx**: 生产/治污工况参数 (如 `p10001` 风机电流)
- **ixxxx**: 辅机状态/参数 (如 `i12001` 运行状态)

## 5. CRC16 校验算法
- **多项式**：`0xA001` (ANSI CRC16)
- **初始值**：`0xFFFF`
- **校验范围**：从 `DataSegment` 的第一个字符开始，到最后一个字符结束（不含包头、长度、包尾）。
- **字节序**：低字节在前，高字节在后 (Little Endian) 或根据 2017 附录 A 示例代码逻辑为准（通常是计算出 int 后转 hex 字符串）。
